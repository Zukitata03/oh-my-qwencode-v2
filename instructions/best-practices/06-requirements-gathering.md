### Round 8-10：非目标和边界

**Round 8**（Contrarian 模式）：
```
目标：挑战假设 | 模糊度：35% → 30%

问题：用户真的需要另一个知识管理工具吗？
     市场已经饱和了（Notion、Roam、Obsidian...）

回答：
竞争分析：
- Notion：太重，适合团队文档
- Roam：双向链接好，但学习曲线陡
- Obsidian：本地优先，不适合协作

我们的定位：
- 轻量 + AI 关联
- 目标用户：知识工作者、研究者
- 差异化：自动关联（减少手动整理）

风险：
- 用户获取成本高
- 需要验证 AI 关联是否真的有价值

缓解：
- 先做 MVP 验证核心价值
- 早期用户访谈迭代
```

**分析**：
- 竞争定位清晰
- 风险识别 + 缓解

**Round 9**（Simplifier 模式）：
```
目标：简化范围 | 模糊度：30% → 25%

问题：如果 MVP 只能做一个功能，是什么？

回答：
核心功能：AI 关联推荐

理由：
- 这是最大差异化
- 其他功能（编辑、链接）都有成熟方案
- AI 关联是"值得做"的理由

简化方案：
- 编辑功能：使用现成 Markdown 编辑器
- 图谱：使用 D3.js 或现成库
- 专注 AI 关联的准确性
```

**分析**：
- 核心价值清晰：AI 关联
- 简化策略：复用现有方案

# 知识管理应用规格

## 价值主张
"让你的知识自己组织起来"
- 帮助用户发现知识之间的隐藏联系

## 目标用户
- 知识工作者
- 研究者
- 学生

## 核心功能（MVP）

### 1. 笔记编辑
- Markdown 编辑器
- 实时预览
- 快速创建（<1 秒）

### 2. 双向链接
- [[笔记名]] 语法
- 反向链接显示
- 链接自动补全

### 3. AI 关联推荐
- 新笔记创建时自动推荐相关笔记
- 基于语义相似度（embedding）
- Top 5 推荐，至少 3 个相关（60% 准确率）
- 用户可反馈"不相关"

### 4. 图谱视图
- 可视化知识网络
- 节点 = 笔记，边 = 链接
- 支持缩放、拖拽
- 100 节点加载 < 2 秒

### 5. 全文搜索
- 支持标题、内容搜索
- 响应 < 500ms
- 支持高亮

- 前端：React + TypeScript
- 后端：Node.js + Express
- 数据库：PostgreSQL
- AI: OpenAI Embedding API
- 部署：Vercel + Railway

## 非目标（v1）

- 移动端 App
- 模板系统
- API 开放
- 第三方集成

- 组件库选择
- AI 模型参数

需要用户确认：

- 域名和品牌

- 图谱加载 < 2 秒
- 搜索响应 < 500ms
