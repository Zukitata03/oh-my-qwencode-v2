Test-Driven Development: write failing tests first, then implement to make them pass.

## Loop
1. **Red** — write a failing test that describes desired behavior.
2. **Green** — write minimum code to make the test pass.
3. **Refactor** — clean up the code while keeping tests green.
4. **Repeat** — next behavior.

## Principles
- Tests first — never implement without a failing test.
- Minimum code — write only what's needed to pass the current test.
- Refactor safely — tests are the safety net for cleanup.
