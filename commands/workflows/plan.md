Activate the **Plan** workflow from oh-my-qwencode.

Create a structured plan before implementation begins.

## Steps
1. Understand the goal — what does "done" look like?
2. Assess current state — explore the codebase for relevant context.
3. Break into ordered steps — dependencies first, then independent work.
4. Define verification criteria — how will we know each step is complete?
5. Identify risks — known unknowns, potential blockers.

## Output
A numbered plan where each step has:
- Clear action description
- Verification criteria (tests, builds, manual checks)
- Estimated complexity (small/medium/large)

After the plan is created, the user can approve it for execution via `$ralph`, `$autopilot`, or `$team`.
